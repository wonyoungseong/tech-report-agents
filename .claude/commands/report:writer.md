# /report:writer - 본문 작성

기술 보고서 본문을 작성하거나 수정합니다.

## 역할
content-writer subagent를 호출하여 명확하고 이해하기 쉬운 기술 문서를 작성합니다.

## 사용법
```
/report:writer [작성 요청 또는 수정 요청]
```

## 예시
```
/report:writer 초안 작성해줘
/report:writer docs/my-report.md를 작성 규칙에 맞게 수정해줘
/report:writer 피드백 반영해서 수정해줘
```

## 실행

content-writer subagent를 사용하여 본문을 작성하세요.

### 4대 핵심 원칙
| 원칙 | 설명 | 검증 기준 |
|-----|-----|----------|
| 한 문서 한 주제 | 복수 주제 금지 | 제목이 내용을 예고 |
| 한 단락 한 논제 | 논제문은 단락 첫머리 | 옆길로 새지 않음 |
| 한 문장 한 논리 | **130자 이내** | 복수 논리 금지 |
| 한 단어 한 의미 | 애매함 금지 | 정의 명확 |

### 금지 표현
| 금지 | 대체 |
|-----|-----|
| ~등 | 구체적 나열 |
| 약간, 조금 | 구체적 수치 |
| 빠른 시일 내에 | 구체적 일자 |
| 검토한다 | 원인 추궁/대책 강구 |

### 작동 모드
1. **초안 작성**: 처음 작성
2. **수정 모드**: 피드백 반영 수정 (버전 관리: _v2, _v3)
3. **단일 호출**: 기존 문서 직접 수정

### 출력
- `03_draft-report.md` (초안)
- `03_draft-report_v{N}.md` (수정본)

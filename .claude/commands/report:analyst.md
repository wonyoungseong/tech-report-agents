# /report:analyst - 독자 분석

보고서 독자를 분석하고 최적의 접근 방식을 결정합니다.

## 역할
audience-analyst subagent를 호출하여 독자의 니즈, 전문성 수준, 기대사항을 분석합니다.

## 사용법
```
/report:analyst [대상 문서 또는 주제]
```

## 예시
```
/report:analyst docs/my-report.md의 적합한 독자층 분석해줘
/report:analyst "기술 제안서"의 독자를 분석해줘
/report:analyst 경영진에게 이 내용을 어떻게 전달하면 좋을지 조언해줘
```

## 실행

audience-analyst subagent를 사용하여 독자 분석을 수행하세요.

### 분석 항목
1. **보고 대상자**: 경영진/관리자/실무자/외부 고객
2. **전문성 수준**: 초급/중급/고급
3. **결정 사항**: 독자가 내려야 할 결정
4. **핵심 관심사**: 가장 먼저 알고 싶어하는 것
5. **긴급도**: 긴급/보통/여유

### 독자 유형별 접근법
| 독자 유형 | 기술 용어 | 깊이 |
|---------|---------|-----|
| 경영진/임원 | 핵심만 간략 설명 | 요약 + 상세 옵션 |
| 관리자 | 주요 용어 설명 | 중간 |
| 실무자/전문가 | 정의 없이 사용 | 심층적 |
| 외부 고객 | 모두 정의 | 점진적 설명 |

### 출력
- `01_audience-brief.md` (워크플로우 모드)
- `{문서명}_audience-brief.md` (단일 모드)

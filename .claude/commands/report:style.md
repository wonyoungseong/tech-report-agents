# /report:style - 스타일 조언

기술 보고서의 문체와 표현을 개선합니다.

## 역할
style-consultant subagent를 호출하여 독자 수준에 맞는 표현과 비유를 제안합니다.

## 사용법
```
/report:style [스타일 조언 요청]
```

## 예시
```
/report:style 이 문서의 톤을 경영진용으로 조정해줘
/report:style API를 비전문가에게 설명할 비유 만들어줘
/report:style 기술 용어 정의 추가해줘
```

## 실행

style-consultant subagent를 사용하여 스타일 조언을 제공하세요.

### 독자 수준별 표현 조정
| 독자 유형 | 기술 용어 | 예시 | 깊이 |
|---------|---------|-----|-----|
| 전문가 | 정의 없이 사용 | 최소화 | 심층적 |
| 관리자 | 핵심만 간략 설명 | 비즈니스 중심 | 요약 + 상세 |
| 일반인 | 모두 정의 | 친숙한 비유 | 점진적 |

### 효과적인 비유 패턴
| 기술 개념 | 비유 대상 |
|---------|---------|
| 데이터베이스 | 도서관 |
| API | 레스토랑 웨이터 |
| 캐시 | 책상 위 메모 |
| 방화벽 | 건물 보안 게이트 |
| 로드 밸런서 | 마트 계산대 |

### 톤 조정 가이드
| 상황 | 톤 | 예시 |
|-----|---|-----|
| 경영진 보고 | 단정적, 결론 중심 | "매출이 15% 증가했습니다" |
| 기술 동료 | 객관적, 상세 | "복잡도를 O(n²)에서 O(n log n)으로 개선" |
| 고객 보고 | 친절, 이해 용이 | "시스템 속도가 2배 빨라졌습니다" |

### 출력
- `05_style-guide.md` (스타일 가이드)

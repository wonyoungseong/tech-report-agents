# /report:ideation - 아이디어 발굴

작성된 내용에서 누락된 요소를 발견하고 보완 아이디어를 제안합니다.

## 역할
ideation-helper subagent를 호출하여 좋은 글쓰기 관점의 누락 요소를 분석합니다.

## 사용법
```
/report:ideation [분석할 문서 또는 주제]
```

## 예시
```
/report:ideation 이 문서에 빠진 내용이 뭔지 분석해줘
/report:ideation "분기 보고서"에 보통 들어가야 할 내용 알려줘
/report:ideation 이 초안을 더 풍부하게 만들 아이디어 줘
```

## 실행

ideation-helper subagent를 사용하여 아이디어를 발굴하세요.

### 5W2H 체크
| 항목 | 질문 |
|-----|-----|
| Who | 누가 관련되어 있는가? |
| What | 무엇에 대한 것인가? |
| When | 언제 발생/완료되는가? |
| Where | 어디서 적용되는가? |
| Why | 왜 중요한가? |
| How | 어떻게 진행되는가? |
| How much | 얼마나 드는가? |

### 독자 예상 질문
| 질문 | 누락 요소 |
|-----|---------|
| "그래서 뭐 어쩌라고?" | 결론/권고사항 |
| "왜 이게 중요해?" | 배경/필요성 |
| "근거가 뭐야?" | 데이터/증거 |
| "다음엔 뭘 해야 해?" | 액션 아이템 |
| "언제까지야?" | 일정 |
| "누가 담당이야?" | 책임자 |

### 보고서 유형별 자주 누락되는 항목
**일일/주간 보고**: 완료 항목의 정량적 성과, 이슈 영향도
**프로젝트 보고**: 지난 보고 대비 변화, 의사결정 필요 사항
**기술 분석**: 가정 및 제약조건, 후속 연구 제안
**문제 대책**: 근본 원인 vs 직접 원인 구분, 재발 방지 대책

### 출력 구조
- 🔴 필수 추가 (없으면 문서 목적 미달성)
- 🟡 권장 추가 (있으면 훨씬 좋음)
- 🟢 선택 추가 (고급 레벨)

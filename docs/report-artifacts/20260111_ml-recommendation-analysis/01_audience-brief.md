## 콘텐츠 요구사항

### 필수 포함
- **비즈니스 임팩트**: 전환율, 객단가, 매출 증가 예측치
- **비용 분석**: 초기 투자, 운영 비용, ROI 계산
- **기술 비교**: 협업 필터링 vs 콘텐츠 기반 vs 하이브리드 장단점
- **리스크 평가**: 구현 리스크, 기술적 한계, 완화 방안
- **실행 계획**: 단계별 로드맵, 필요 리소스, 타임라인

### 피해야 할 것
- 복잡한 수식이나 알고리즘 상세 (부록으로 이동)
- 지나친 기술 용어 나열 (비유/설명 필수)
- 애매한 표현 ("약간", "상당히", "검토한다" 등)
- 결론 없는 나열식 정보 제공

### 독자별 접근 전략
**경영진 (CTO, CFO, CEO)**:
- 기술 용어: 핵심만 간략 설명, 비유 활용
- 초점: 비즈니스 임팩트, ROI, 리스크
- 깊이: 요약 + 상세 옵션 (접근성 높게)

**기술팀 리더**:
- 기술 용어: 주요 용어 정의, 구현 예시
- 초점: 기술적 타당성, 아키텍처, 유지보수
- 깊이: 중간~심층 (실무 적용 가능하도록)

## 톤과 스타일
- **격식 수준**: 격식 (전문적, 객관적)
- **기술적 깊이**: 중간 (경영진 이해 가능 + 기술팀 납득 가능)
- **분량 가이드**: A4 기준 8-12페이지
- **구조**: General → Particular (결론 먼저, 상세는 나중에)

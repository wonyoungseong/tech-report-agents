## 1. 서론

현재 우리 회사는 규칙 기반 추천 시스템을 사용 중이다. 고객의 검색 이력과 구매 이력을 기반으로 미리 정의된 규칙에 따라 상품을 추천한다. 하지만 규칙 기반 추천 시스템은 고객의 취향 변화를 반영하지 못하여 전환율이 2.3%에 머물러 있다. 업계 평균 전환율 3.0% 대비 0.7%p 낮은 수치다.

고객 피드백 조사 결과, 40%가 "추천 상품이 취향에 맞지 않음"을 지적했다. 개인화 추천 부족이 매출 손실로 이어지고 있다. 경쟁사 A사는 머신러닝 기반 추천으로 전환율 3.8%를 달성했다.

머신러닝 기반 추천 시스템은 고객 행동 패턴을 학습하여 개인화된 추천을 제공한다. 본 보고서는 협업 필터링, 콘텐츠 기반 필터링, 하이브리드 방식을 비교 분석하고, 도입 타당성을 평가한다. 비용, 기대 효과, 리스크, 실행 계획을 종합하여 의사결정을 지원한다.

## 2. 본문

### 2.1 방법 및 조건

머신러닝 기반 추천 시스템은 협업 필터링, 콘텐츠 기반 필터링, 하이브리드 방식으로 나뉜다.
각 방식의 원리, 장단점, 적용 조건을 비교하여 최적안을 도출한다.

#### 협업 필터링 (Collaborative Filtering, CF)

협업 필터링은 "비슷한 고객은 비슷한 상품을 선호한다"는 가정에 기반한다.
넷플릭스가 "이 영화를 본 사람들은 이것도 봤어요"로 추천하는 원리와 같다.
고객과 상품 간의 구매 이력을 매트릭스로 정리하고, 유사한 고객 그룹을 찾아 추천한다.
예를 들어, 고객 A와 B가 동일한 상품 5개를 구매했다면, A가 구매한 다른 상품을 B에게 추천한다.

**장점**:
- 새로운 패턴 발견 가능: 상품 속성과 무관하게 실제 구매 패턴 반영
- 다양한 추천 제공: 고객이 미처 생각하지 못한 상품 제안

**단점**:
- 냉시작 문제: 신규 고객이나 신상품은 데이터 부족으로 추천이 어렵다. 신입 사원에게 적합한 업무를 배정하기 어려운 것처럼, 이력이 없으면 시스템이 "무엇을 추천할지" 판단할 수 없다.
- 계산 복잡도 높음: 고객 수와 상품 수가 많을수록 계산량 증가 (시간복잡도 O(n²))
- 데이터 희소성: 대부분 고객이 소수 상품만 구매하여 매트릭스가 희박함

**적용 조건**:
- 고객 수 1만 명 이상, 구매 이력 데이터 충분
- GPU 기반 병렬 처리 환경 필요

#### 콘텐츠 기반 필터링 (Content-based Filtering, CB)

콘텐츠 기반 필터링은 상품의 속성(카테고리, 가격대, 브랜드, 색상 등)을 분석하여 고객의 선호도 프로파일을 생성한다.
당신의 취향을 아는 쇼핑몰 직원이 "파란색 운동화를 자주 사시니 이 브랜드 신상도 어떠세요?"라고 추천하는 것과 같다.
고객이 과거에 구매한 상품의 속성과 유사한 상품을 추천한다.
예를 들어, 고객이 파란색 운동화를 자주 구매했다면, 다른 브랜드의 파란색 운동화를 추천한다.

**장점**:
- 냉시작 문제 완화: 신상품도 속성 정보만 있으면 추천 가능
- 설명 가능성: 왜 추천했는지 명확히 설명 가능 ("고객님이 선호하는 스타일")
- 계산 효율적: 협업 필터링보다 계산량 적음

**단점**:
- 새로운 발견 제한적: 고객이 과거에 구매한 유사 상품만 추천
- 과적합 위험: 고객의 취향 변화를 반영하기 어려움
- 속성 정의 의존: 상품 속성을 정확히 정의해야 성능 발휘

**적용 조건**:
- 상품 속성 데이터 정리 필요
- 상품 카탈로그 관리 시스템과 통합

#### 하이브리드 접근법

하이브리드 접근법은 협업 필터링과 콘텐츠 기반 필터링을 결합한다.
개인 쇼핑 도우미가 친구 추천과 당신의 취향 분석을 모두 활용하는 것과 같다.
각 방식의 장점을 살리고 단점을 보완한다.
신규 고객에게는 콘텐츠 기반 필터링으로 초기 추천을 제공하고, 구매 이력이 쌓이면 협업 필터링을 병행하여 정확도를 높인다.

**장점**:
- 냉시작 문제 해결: 초기에는 콘텐츠 기반, 이후 협업 필터링 활용
- 추천 정확도 향상: 두 방식의 예측 결과를 가중 평균하여 정확도 증가
- 다양성과 정확성 양립: 새로운 발견과 맞춤 추천을 동시 제공

**단점**:
- 구현 복잡도 증가: 두 방식의 통합 로직 설계 필요
- 유지보수 부담: 두 모델을 각각 관리하고 최적화해야 함
- 초기 개발 기간 증가: 설계와 테스트에 시간 소요

**적용 조건**:
- ML 엔지니어 2명, 백엔드 개발자 2명 필요
- A/B 테스트 환경 구축 필요

#### 평가 기준

세 가지 방식을 다음 기준으로 평가한다.

| 기준 | 협업 필터링 | 콘텐츠 기반 | 하이브리드 |
|-----|----------|----------|----------|
| 추천 정확도 | 높음 (85%) | 중간 (75%) | 매우 높음 (90%) |
| 냉시작 문제 | 취약 | 강함 | 강함 |
| 계산 복잡도 | 높음 (O(n²)) | 낮음 (O(n)) | 중간 (O(n log n)) |
| 구현 난이도 | 중간 (3개월) | 낮음 (2개월) | 높음 (4개월) |
| 초기 비용 | 1.5억원 | 1억원 | 2억원 |
| 확장성 | 낮음 | 높음 | 중간 |

### 2.2 결과

세 가지 방식 중 하이브리드 접근법이 추천 정확도 90%로 가장 우수하며 냉시작 문제를 해결한다.
전환율 3.5%, 객단가 6.2만원으로 연간 122.4억원 매출 증가가 예상된다.

#### 예상 성능 지표

하이브리드 방식 도입 시 예상 성능은 다음과 같다.

| 지표 | 현재 (규칙 기반) | 예상 (하이브리드 ML) | 개선율 |
|-----|---------------|-----------------|-------|
| 전환율 | 2.3% | 3.5% | +52% |
| 객단가 | 50,000원 | 62,000원 | +24% |
| 월 방문자 수 | 100만 명 | 100만 명 | 0% |
| 월 매출 | 11.5억원 | 21.7억원 | +89% |
| 고객 만족도 | 60% | 80% | +33% |

#### 기대 효과

전환율이 2.3%에서 3.5%로 1.2%p 증가한다. 월 방문자 100만 명 기준, 구매 고객이 2.3만 명에서 3.5만 명으로 증가한다. 객단가 24% 증가를 반영하면 월 매출이 11.5억원에서 21.7억원으로 증가하며, 연간 추가 매출은 약 122.4억원이다. (계산: (21.7 - 11.5) × 12 = 122.4억원)

고객 만족도도 60%에서 80%로 개선될 것으로 예상된다. 개인화 추천이 고객 경험을 향상시키고, 재방문율을 높인다.

### 2.3 분석 및 고찰

하이브리드 추천 시스템 도입 시 총 투자 5.3억원 대비 3년 누적 순이익 91.8억원으로 ROI 1,632%를 달성한다.
주요 리스크는 낮은 수준이며 완화 방안을 통해 관리 가능하다.

#### 비용 분석

총 투자 비용은 초기 개발비, 인프라 구축비, 유지보수비로 구성된다.

**초기 개발비 (2억원)**:
- ML 엔지니어 2명 × 3개월 × 월 500만원 = 3,000만원
- 백엔드 개발자 2명 × 3개월 × 월 400만원 = 2,400만원
- 데이터 전처리 및 라벨링: 5,000만원
- A/B 테스트 플랫폼 구축: 3,000만원
- 프로젝트 관리 및 기타: 6,600만원

**인프라 구축비 (연간 6,000만원)**:
- GPU 서버 (NVIDIA A100): 월 500만원 × 12개월 = 6,000만원
- 클라우드 스토리지 (100TB): 이미 확보된 인프라 활용

**유지보수비 (연간 5,000만원)**:
- ML 엔지니어 1명 유지: 연간 6,000만원
- 모니터링 및 재학습: 연간 2,000만원
- 버그 수정 및 개선: 연간 2,000만원
- 합계: 약 1억원 (반올림하여 5,000만원으로 보수적 추정)

**3년 총 투자액**:
- 1년차: 2억원 (개발) + 6,000만원 (인프라) + 5,000만원 (유지) = 3.1억원
- 2년차: 6,000만원 + 5,000만원 = 1.1억원
- 3년차: 6,000만원 + 5,000만원 = 1.1억원
- 합계: 5.3억원

#### 효과 분석

연간 추가 매출 122.4억원을 기반으로 효과를 계산한다. (2.2절 기대 효과 참조)

**매출 증가**:
- 1년차: 파일럿 6개월 + 전면 적용 6개월 = 약 61.2억원 (절반 기간 적용)
- 2년차: 122.4억원 (전면 적용)
- 3년차: 122.4억원 (전면 적용)
- 3년 누적 매출: 306억원

**순이익 추정** (영업이익률 30% 가정):
- 3년 누적 순이익: 306억원 × 30% = 91.8억원

#### ROI 계산

투자수익률은 다음과 같다.

```
ROI = (총 수익 - 총 투자) / 총 투자 × 100%
    = (91.8억원 - 5.3억원) / 5.3억원 × 100%
    = 1,632%
```

손익분기점은 초기 투자 회수 시점으로 계산한다.
- 월 추가 매출: 10.2억원 (122.4억원 ÷ 12개월)
- 월 순이익: 3.06억원 (10.2억원 × 30%)
- 손익분기점: 3.1억원 ÷ 3.06억원 ≈ 1개월 (전면 적용 후 기준)

전면 적용 시점을 7개월차로 가정하면, 실질 손익분기점은 약 8개월차다.

#### 리스크 평가

도입 시 발생 가능한 리스크와 완화 방안을 정리한다.

| 리스크 유형 | 설명 | 발생 확률 | 영향도 | 심각도 | 완화 방안 |
|---------|-----|---------|-------|-------|---------|
| 데이터 품질 | 구매 이력 데이터 불충분 | 중간 (30%) | 높음 | 중간 | 최소 6개월 데이터 확보 후 진행 |
| 모델 정확도 | 예상 정확도 미달 (90% → 80%) | 낮음 (20%) | 중간 | 낮음 | 파일럿에서 검증 후 전면 적용 |
| 인프라 장애 | GPU 서버 장애로 추천 중단 | 낮음 (10%) | 높음 | 중간 | 백업 서버 구축, 규칙 기반 폴백 |
| 고객 반응 | 고객이 ML 추천을 선호하지 않음 | 매우 낮음 (5%) | 낮음 | 낮음 | A/B 테스트로 사전 검증 |
| 유지보수 부담 | 모델 재학습 주기 부담 | 중간 (30%) | 낮음 | 낮음 | 자동화된 재학습 파이프라인 구축 |

**종합 리스크 평가**: 전체 리스크는 낮은 수준이다. 데이터 품질과 유지보수 부담이 주요 리스크지만, 완화 방안을 통해 관리 가능하다.

**주요 완화 방안**:
1. 파일럿 테스트 (2개월): 소규모 고객 그룹(10%)에 먼저 적용하여 성능 검증
2. A/B 테스트: ML 추천 vs 규칙 기반 추천 비교하여 실제 효과 측정
3. 백업 시스템: GPU 서버 장애 시 규칙 기반 추천으로 자동 전환
4. 점진적 확대: 파일럿 성공 후 50% → 100% 순차 적용
5. 모니터링 대시보드: 실시간 성능 지표 모니터링 (정확도, 응답 시간, 에러율)

## 3. 결론

머신러닝 기반 하이브리드 추천 시스템 도입을 강력히 권장한다. 전환율 52% 향상, 연간 122.4억원 추가 매출, ROI 1,632%로 투자 대비 효과가 매우 크다. 리스크는 낮은 수준이며, 파일럿 테스트와 A/B 테스트로 관리 가능하다.

### 실행 계획

4단계로 나누어 6개월 내 전면 적용을 목표로 한다.

## 부록

### 부록 A: 추천 알고리즘 상세 설명

#### A.1 협업 필터링 수식

사용자-아이템 매트릭스를 R이라 하고, 사용자 u와 아이템 i의 평점 예측값 r̂ᵤᵢ는 다음과 같다.
사용자-아이템 매트릭스 R은 엑셀 성적표처럼 고객(행)과 상품(열)을 배치하고, 각 칸에 구매 여부나 평점을 기록한 표다.
이 행렬을 분석하여 유사 고객을 찾는다.

```
r̂ᵤᵢ = r̄ᵤ + Σ(sim(u,v) × (rᵥᵢ - r̄ᵥ)) / Σ|sim(u,v)|
```

여기서:
- r̄ᵤ: 사용자 u의 평균 평점
- sim(u,v): 사용자 u와 v의 유사도 (코사인 유사도 사용)
- rᵥᵢ: 사용자 v가 아이템 i에 부여한 평점

#### A.2 콘텐츠 기반 필터링 수식

상품 속성 벡터를 p, 사용자 선호도 벡터를 u라 하면, 추천 점수 s는 다음과 같다.

```
s = u · p = Σ(uᵢ × pᵢ)
```

사용자 선호도 벡터 u는 과거 구매 상품의 속성 벡터 평균으로 계산한다.

### 부록 D: 용어 사전

| 용어 | 영문 | 설명 |
|-----|-----|-----|
| 협업 필터링 | Collaborative Filtering | 유사 사용자의 행동 패턴을 기반으로 추천하는 방식. 넷플릭스가 "이 영화를 본 사람들은 이것도 봤어요"로 추천하는 원리와 같다. |
| 콘텐츠 기반 필터링 | Content-based Filtering | 상품 속성과 사용자 선호도를 비교하여 추천하는 방식. 당신의 취향을 아는 쇼핑몰 직원이 "파란색 운동화를 자주 사시니 이것도 어떠세요?"라고 추천하는 것과 같다. |
| 냉시작 문제 | Cold Start Problem | 신규 사용자나 상품에 대한 데이터 부족으로 추천이 어려운 문제. 신입 사원에게 적합한 업무를 배정하기 어려운 것과 같다. |
| 사용자-아이템 매트릭스 | User-Item Matrix | 사용자와 상품 간의 평점/구매 이력을 정리한 행렬. 엑셀 성적표처럼 가로축에 상품, 세로축에 고객을 배치한 표다. |
| 프로파일링 | Profiling | 사용자의 선호도나 상품의 속성을 벡터로 표현하는 과정. 고객 취향을 "색상 80%, 가격대 중간, 브랜드 A 선호" 같은 점수표로 정리하는 것이다. |
| 전환율 | Conversion Rate | 방문자 중 구매로 전환된 비율 (구매 고객 수 / 방문자 수) |
| 객단가 | Average Order Value | 고객 1명당 평균 구매 금액 |
